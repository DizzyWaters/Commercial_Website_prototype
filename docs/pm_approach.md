# Project Management Approach

This project will use a lightweight hybrid approach based on Kanban and Scrum principles.
Kanban will be used to track ongoing tasks such as feature implementation, bug fixing, testing, and documentation.
Scrum-style planning will be used to divide the project into short development stages, each focused on delivering a usable part of the MVP.

The human developer is responsible for product decisions, architecture, code review, final testing, and deciding whether AI-generated changes are acceptable. The AI assistant may help with boilerplate code, unit tests, documentation drafts, refactoring suggestions, and small implementation tasks. AI-generated output must be reviewed before it is accepted into the project, especially when it affects architecture, security, data handling, or business logic.
